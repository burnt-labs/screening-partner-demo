Screening by Burnt · Partner API Demo

Provision a unit & screening → start a no-login applicant screening → poll the household decision.
Proxying to … API key: server-side only ✓
The browser never sees the API key — it calls this local proxy, which adds the Authorization: Bearer header server-side. Complete a screening in the Burnt app (via the apply_url below), then poll section 3 until a decision lands.

1 Create unit + screening

POST /api/v1/units. Only property_label + monthly_rent_cents are required. Package total is a flat $20.

Uncheck all to create a bare unit (no screening; application comes back null).
Required when the landlord owes > $0 — i.e. operator mode, or applicant with pays < $20.

2 Start no-login screening

POST /api/v1/units/{unitId}/screenings. Only applicant_email is required. Re-calling with the same external_id re-issues the same screening.

3 Poll application-group status

GET /api/v1/application-groups/{id}. No PII — status, decision, and per-member verification only.

✓ Webhooks received (server)

Verified events this proxy has received at /webhooks/burnt. Usually empty locally — see the README (needs a public HTTPS tunnel + dashboard config; the group-completed event rides a cron that doesn't run under local dev). Polling above is the reliable local path.
